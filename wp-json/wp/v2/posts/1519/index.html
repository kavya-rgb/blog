\n Best time to sort?
\n<\/h2>\n

By definition, sorting seems obvious\u2013 given text or numerical data, if you display text data in alphabetical order, you\u2019re sorting by ascending values. Conversely, if you display the biggest to smallest numbers, you\u2019re sorting by descending values.<\/p>\n

Rather than asking what sorting is, the worthier question is whether your data pre-processing is optimized by sorting specific columns. Thus, what kinds of insights are you searching for in your data? In which situations, with what type of data, does sorting columns benefit your analysis? <\/p>\n

Below are scenarios described by our co-founders where it was immensely useful to leverage sorting:<\/p>\n

    \n
  • \n

    Viewing a user\u2019s first and last page visit: when given user traffic data on a website (like our Mage blog) we can sort by timestamp<\/strong> per user. The first page tells us which page brought them to the site. The last page would say what made them drop off, and hopefully it\u2019s not one of my articles \ud83d\ude1b <\/p>\n<\/li>\n

  • \n

    Discern what kind of content users like to consume in the morning: As a newsfeed app, if we sort by timestamp<\/strong> per user, we can gather a list of articles they \u2764\ufe0f to store in a column. With this data, we can train a model that decides what to show the user in the morning and keep them engaged with the app. <\/p>\n<\/li>\n<\/ul>\n

    As shown in these examples, sort is rarely used on its own, and it\u2019s most powerful when used with other operations, like to group by<\/strong> user. <\/p>\n
