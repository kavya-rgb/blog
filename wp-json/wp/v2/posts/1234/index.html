In this LeetCode challenge<\/a> we\u2019re asked to find two numbers in a given array which add up to make a specific number. So in other words, given the array [1, 2, 3]<\/code> and a target number of 5<\/code> , we would return [2, 3]<\/code>.<\/p>\n

Disclaimer:<\/strong> We\u2019re actually asked to return the indices of the numbers, not the numbers themselves, but the above made for a much simpler explanation!<\/p>\n

This challenge has a variety of solutions, and all revolve more or less around the same concept: For each number in the provided array, check if the complement to each number exists, and if so return the number and its complement.<\/p>\n

Wondering what the complement is? In this case, it means \u201cThe number that, when added to the current number, makes the target number\u201d. So for example, if you have 3 and need 5, your complement will be 2, because 5 – 3 = 2.<\/p>\n

\n
\n <\/a>
\n Solution 1: Nested for loop
\n<\/h2>\n

Here\u2019s a super straightforward example used a nested loop:<\/p>\n

In this example we loop through all of the numbers, and then for each number, we once again loop through all of the numbers (excluding the first one, as the question states we cannot use the same number twice), and check if the second number is the required complement to the first one.<\/p>\n

This solution works fine, but nested loops like this are slow and generally frowned upon, so let\u2019s look at some other approaches.<\/p>\n

\n
\n <\/a>
\n Solution 2: Maps
\n<\/h2>\n

In this example we declare an empty Map and then loop through the array of numbers. For each number, we check if the required complement exists in the Map, returning both numbers if so, or we add the current number into the Map. By the time the loop has reached the end of the array, we then have a map of all values provided (assuming the solution hasn\u2019t already been found).<\/p>\n

Maps are incredibly quick for looking up data, so this solution is much faster than the nested for loop approach, but surprisingly it\u2019s not the most common solution I see to this type of answer.<\/p>\n

\n
\n <\/a>
\n Solution 3: HashMap \/ Object lookups
\n<\/h2>\n

Note:<\/strong> In JavaScript, Objects are not<\/em> straight up HashMaps, but they\u2019re often referred to as such.<\/p>\n

In what is a near-identical solution to the Map approach, this method stores the numbers one-by-one in an object, which then allows the program to lookup the value using an object key, rather than looping. This means that \u2014 again similar to Maps \u2014 we can look up our complement, rather than looping through all of the data to find it.<\/p>\n
