\n How to Automatically deploy from GitHub (Git) to cPanel
\n<\/h2>\n

This tutorial provides a quick way of deploying files from GitHub to cPanel. We are only deploying from Git(Hub); thus making GitHub our single source of truth. In other words, we will be pulling<\/em> from GitHub.<\/p>\n

\n

cPanel<\/a> is an online Linux-based graphical user interface (GUI) used as a control panel to simplify website and server management. cPanel allows you to publish websites, manage domains, organize web files, create email accounts, and even more. cPanel is one of the popular control panels.<\/p>\n<\/blockquote>\n

In this tutorial, I shall walk you through configuring and setting up your Git repo and have it auto deploy to cPanel. Let’s get started. First, you need to login into cPanel.<\/p>\n

\n
\n <\/a>
\n Login in into cPanel
\n<\/h2>\n

To login to cPanel, on your browser, navigate to https:\/\/yoursite.com\/cpanel<\/code> or https:\/\/yoursite.com:2083<\/code>.<\/p>\n

\n
\n <\/a>
\n 1. Navigate to Git Version Control
\n<\/h2>\n

To navigate to Git Version Control, follow cPanel<\/strong> > Home<\/strong> > File<\/strong> > Git Version Control<\/strong>. The GVC allows you to configure deployment for your cPanel-managed repositories.<\/p>\n

<\/a><\/p>\n

\n
\n <\/a>
\n 2. See or Link your Repositories
\n<\/h2>\n

The next screen that you will see will enable you manage your repositories. I have a repository already linked from GitHub to cPanel. If you currently do not have any repo linked, then go ahead and click on “Create<\/strong>” .<\/p>\n

\n

If you already have a repo from Git link to your cPanel, please skip to Step 4<\/strong><\/p>\n<\/blockquote>\n

<\/a><\/p>\n

\n
\n <\/a>
\n 3. Clone your Repository
\n<\/h2>\n

This step will enable you to clone your remote repo to cPanel. You should fill in the fields such as clone URL<\/strong>, Repository Path<\/strong>.<\/p>\n

    \n
  1. \n

    The Clone URL<\/em> refers to your remote repository URL. It must begin with either http:\/\/<\/code><\/em>, https:\/\/<\/code><\/em>, ssh:\/\/<\/code><\/em> or git:\/\/<\/code><\/em> Example<\/strong>: https:\/\/github.com\/Eunit99\/eunit99.git<\/a><\/p>\n<\/li>\n

  2. \n

    The Repository Path<\/em> refers to directory where your repo will live on cPanel. The path is usually home\/YOUR-USERNAME\/REPOSITORY-NAME<\/code>
    If everything looks good, go ahead and click on “Create<\/strong>“<\/p>\n<\/li>\n<\/ol>\n

    <\/a><\/p>\n

    \n
    \n <\/a>
    \n 4. Manage Repositories
    \n<\/h2>\n

    This step will allow us manage our repositories which we have on cPanel. To navigate to Git Version Control, follow cPanel<\/strong> > Home<\/strong> > File<\/strong> > Git Version Control<\/strong>. The GVC allows you to configure deployment for your cPanel-managed repositories.<\/p>\n

    You will see a list of all repositories currently linked to cPanel from Git. Click on Manage<\/strong> to be taken to the next step.<\/p>\n

    <\/a><\/p>\n

    \n
    \n <\/a>
    \n 5. Pull or Deploy
    \n<\/h2>\n

    After having filled the fields for information about your repos in step 4 above, now is the time to clone your remote repo to cPanel just like how you would on your local machine \ud83d\ude0a<\/p>\n

    \n

    Pull deployment pulls changes from a remote repository to your cPanel and deploys your new changes live.<\/p>\n<\/blockquote>\n

      \n
    1. Click on Pull or Deploy<\/strong> tab<\/li>\n
    2. Click on Update from Remote<\/strong>\n<\/li>\n
    3. Click on Deploy HEAD Commit<\/strong>\n<\/li>\n<\/ol>\n

      <\/a><\/p>\n

      \n
      \n <\/a>
      \n \ud83c\udf86\u2728Congratulations! You have successfully deployed a remote repo to cPanel
      \n<\/h2>\n

      \ud83d\ude32 But we wanted an auto deployment of new changes from GitHub to cPanel, isn’t it? If yes, then keep reading.<\/p>\n

      \n
      \n <\/a>
      \n 6. Setting up auto-deployment
      \n<\/h2>\n

      In order to deploy changes from a cPanel-managed repository, you must have a .cpanel.yml<\/code> file in your top-level directory of your repository. Create a .cpanel.yml<\/code> file and add this code below:<\/p>\n

      <\/a><\/p>\n
