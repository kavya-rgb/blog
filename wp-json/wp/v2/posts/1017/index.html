\n The findOrFail method also accepts a list of ids
\n<\/h3>\n

The findOrFail method also accepts a list of ids. If any of these ids are not found, then it fails. Nice if you need to retrieve a specific set of models and don’t want to have to check that the count you got was the count you expected<\/p>\n

\n
User::create(['id' => 1]);\nUser::create(['id' => 2);\nUser::create(['id' => 3]);\n\n\/\/ Retrives the user...\n$user = User::findOrFail(1);\n\n\/\/ Throws a 404 because the user doesn't exist...\nUser::findOrFail(99);\n\n\/\/ Retrives all 3 users...\n$users = User::findOrFail([1, 2, 3]);\n\n\/\/ Throws because it is unable to find *all* of the users\nUser::findOrFail([1, 2, 3, 99]);\n<\/code><\/pre>\n
