I\u2019ve been using Postman since its inception and have enjoyed watching it turn into an incredible and ever-growing API platform<\/a>. But why did I recently decide to officially join the company as Postman OpenAPI Technical Lead? Because Postman doesn\u2019t just help people use APIs, it\u2019s committed to helping people understand and create APIs.<\/p>\n

\n
\n <\/a>
\n The Handyman who fixes APIs
\n<\/h2>\n

I\u2019m Arnaud Lauret, the Paris-based author of The Design of Web APIs<\/em>, and some in the API community know me better as the API Handyman. That\u2019s because I think of APIs as my trade; I love working on all aspects of APIs, but I\u2019m especially interested in design and governance and have always been fond of the OpenAPI Specification<\/a>. For 20 years, I have connected pieces of software together or helped people do so. Over the last seven of those years, I\u2019ve been sharing on my blog, at conferences, or in my book all that I struggle with, learn, and teach while working in the API space.<\/p>\n

During all these years of practice, failures, and successes, I\u2019ve learned so much, but I\u2019m still learning\u2014about architecture, heterogeneous distributed systems, decade(s)-old legacy or brand-new systems APIfication, API design, API governance, taking advantage of APIs from a business perspective, creating API products, and, most recently, becoming API-first<\/a>. Working with businesses and IT, I have been helping dozens of teams better understand and create APIs, promoting the API-first vision. I\u2019ve worked on design guidelines and governance, conducted hundreds of API design reviews, and I\u2019ve trained hundreds of people.<\/p>\n

Whatever I do, create, use, learn, or discover along the way, I always think about how to share it with others so that they can gain knowledge from my, or other people\u2019s, experiences. And I like to apply some core API principles when doing that: make things easy to understand, easy to use, easy to reuse, and easy to evolve. My motto as the API Handyman has been \u201chelping people understand and create APIs.\u201d Joining Postman will allow me to take this motto to the next level.<\/p>\n

\n
\n <\/a>
\n Helping more people use, understand, and create APIs with Postman
\n<\/h2>\n

Postman was created to help people use APIs<\/a>, and I can say it helped me a lot. I remember fondly how I started to use the Postman Chrome extension in 2012 to discover and test APIs, then to create and share collections so that others can easily use the APIs I was creating or using. Now Postman, both the company and that tool that has since become a comprehensive platform<\/a>, is supporting a global community of more than 17 million users in so many ways.<\/p>\n

This is possible due to the ever-growing number of enhancements and new features<\/a> of the Postman platform and also thanks to the company\u2019s API-first vision. I\u2019m excited about joining the Postman Open Technologies<\/a> team of API thought leaders and being able to study and contribute to the API space by working on design, governance, and OpenAPI. I hope to make knowledge on these subjects easily accessible, all to make API successes easily repeatable.<\/p>\n

\n
\n <\/a>
\n See you space APIers
\n<\/h2>\n

I already have plenty of API stories to tell but I look forward to hearing from you: the more questions you ask, and the more you share, the more Postman will be able to help you and our greater community.<\/p>\n

If you\u2019re interested in APIs, helping others, and creating an awesome API product, why not become a Postmanaut yourself? Wherever you are on the global map, check out Postman\u2019s Careers page<\/a> to see what roles might be a fit with your skills.<\/p>\n
