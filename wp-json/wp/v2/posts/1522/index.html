\n<\/p>\n

\n

Among major programming languages, there are probably none that feel scarier for beginners than Haskell. And some years ago, it was so for a reason.<\/p>\n

I first attempted to learn Haskell in 2017 and failed miserably. There were several causes: me not being very experienced with programming, a lack of materials that would fit my learning style, and the confusing Haskell ecosystem. <\/p>\n

But I can happily say that learning Haskell has become much easier now due to continuous work done by the community on improving the learning resources and ecosystem. <\/p>\n

If you want to learn Haskell and finally understand what people mean by monads in all those Hacker News comments, it\u2019s possible. To help you, I\u2019ve put together a list of best Haskell beginner resources along with short descriptions of them. <\/p>\n

\n
\n <\/a>
\n \ud83e\udd14 Why Haskell?
\n<\/h2>\n
\n

“A language that doesn’t affect the way you think about programming is not worth knowing.” \u2013 Alan Perlis<\/p>\n<\/blockquote>\n

Learning Haskell has many benefits. It\u2019s a decent general-purpose language, and there are some areas such as blockchain and fintech where Haskell and other typed FP languages are frequently used. <\/p>\n

At the same time, you have to understand that Haskell is neither extremely popular nor widely demanded in the job market. Especially for Haskell juniors, it can be hard to find a position writing Haskell. <\/p>\n

Therefore, I think the most significant benefit of learning a language like Haskell is that you\u2019ll expand your horizon and become a better programmer. The concepts you learn in Haskell will help you write better code in other programming languages, even in conventional ones like JavaScript, Python, or Ruby. <\/p>\n

\n
\n <\/a>
\n \ud83d\uddfa\ufe0f How to learn Haskell?
\n<\/h2>\n

First step: close that monad tutorial. \ud83d\ude43<\/p>\n

Here\u2019s what I would do: <\/p>\n

    \n
  • To begin, watch one of the courses listed in the course section<\/a> of this article to get a feel for the language. <\/li>\n
  • After that, you can use one of the books<\/a> I\u2019ve listed to structure your learning process.<\/li>\n
  • At the same time, start building your own projects (CLI todo tool, web server) or doing exercises in Exercism<\/a> or CodinGame<\/a>. <\/li>\n
  • Keep increasing your knowledge, looking up things you need for your projects, and building small applications to test what you learn.<\/li>\n
  • If all works out well, you should be able to do basic tasks with Haskell. Then, you can turn to resources like Haskell in Depth<\/a> or just explore the areas of Haskell that interest you by reading blog posts and talking to other Haskellers.<\/li>\n<\/ul>\n

    \n
    \n <\/a>
    \n \ud83d\udcd6 Haskell books
    \n<\/h2>\n

    For your first book, I suggest Haskell Programming from First Principles<\/a> or Get Programming With Haskell<\/a>. The first is better if you are interested in building up your Haskell knowledge from the bottom up. The second is better if you would like to start writing programs in Haskell as soon as possible.<\/p>\n

    \n
    \n <\/a>
    \n Haskell Programming from First Principles
    \n<\/h3>\n

    If you\u2019re searching for the book<\/strong> that will contain everything you need to start writing Haskell, Haskell Programming from First Principles<\/a> by Christoph Allen and Julie Moronuki is the choice. <\/p>\n

    It\u2019s very exhaustive and will build your Haskell knowledge from the ground up, starting with lambda calculus and basic Haskell syntax, then covering all the common typeclasses, monads such as State and Reader, monad transformers, and IO. The explanations are very beginner-friendly, and there are plenty of exercises to try out your new knowledge. <\/p>\n

    At the same time, it\u2019s a little bit too verbose for my taste and sometimes feels more like a lexicon\/reference than a beginner programming book. <\/p>\n

    \n
    \n <\/a>
    \n Get Programming with Haskell
    \n<\/h3>\n

    This is the book that I\u2019d personally suggest. <\/p>\n

    In contrast to HPFP, Get Programming with Haskell<\/a> will help you make usable programs before delving into the theory of functors, monads, etc. It follows a more practical approach, covering only the essentials and then moving on to show you how to build things like small command-line applications, work with texts, IO, glitch JPEGs, and other fun stuff. <\/p>\n

    \n
    \n <\/a>
    \n Learn You a Haskell for Great Good!
    \n<\/h3>\n

    As a commonly recommended resource, Learn You a Haskell<\/a> deserves mention here as well.<\/p>\n

    It might be a good third choice if the books above don\u2019t suit you and you are dead set on reading a book, but I would generally recommend avoiding it.<\/p>\n

    Unfortunately, one can feel that time has passed since the release. Some of the code examples supposedly don’t work anymore, and the official website is sometimes down. In addition, it doesn\u2019t contain any exercises.<\/p>\n

    There have been some community-organized efforts to create better forks of it, but nothing substantial. <\/p>\n

    \n
    \n <\/a>
    \n \ud83d\udc68\u200d\ud83c\udfeb Haskell courses
    \n<\/h2>\n

    If reading books isn\u2019t exactly your thing, here are some Haskell courses that should get you writing Haskell as fast as possible. There are two options to choose from here: something like Type Classes<\/a> or learn4haskell<\/a> can be very useful for getting started with the Haskell syntax and basic language constructs, while the more university-focused courses from Erik Meijer<\/a> or Graham Hutton<\/a> will help you start thinking like a functional programmer.<\/p>\n

    \n
    \n <\/a>
    \n Type Classes
    \n<\/h3>\n

    Type Classes<\/a> is a collection of courses created by Julie Moronuki and Chris Martin. Julie previously worked on Haskell Programming from First Principles, a book I mentioned as a good choice for beginners. <\/p>\n

    I believe these courses are even better than the book. As far as I saw, the material is well structured and accommodates different styles of learning by having lessons in both video and text. New courses are regularly added to the website. There\u2019s just one downside: the rather costly subscription fee.<\/p>\n

    But don\u2019t worry. The most important Type Classes course for beginners is the beginner crash course<\/a>, which you can view for free.<\/p>\n

    \n
    \n <\/a>
    \n Haskell Beginners 2022 and learn4haskell
    \n<\/h3>\n

    Dmitrii Kovanikov has published video recordings from his free Haskell course for beginners, which you can find on his YouTube channel<\/a>. The course is short and sweet, covering FP, syntax, data types, functors, and monads in just 4 hours. <\/p>\n

    It covers more or less the same ground as learn4haskell<\/a> \u2013 a GitHub-based course that helps you learn Haskell by making 4 pull requests \u2013 which Dmitrii did together with Veronika Romashkina.<\/p>\n

    If your goal is to get to writing monads as quickly as possible, one of these could be the best possible choice. Both the YouTube course and the GitHub course are very accessible and quite fun. <\/p>\n

    \n
    \n <\/a>
    \n Introduction to Functional Programming
    \n<\/h3>\n

    If you would like to take a fundamentals-focused approach towards learning Haskell, Introduction to Functional Programming<\/a> could be the choice for you. It is an online course taught by Erik Meijer that uses Haskell to teach you how to think about programs and write them in any functional programming language.<\/p>\n

    \n
    \n <\/a>
    \n Functional Programming in Haskell
    \n<\/h3>\n

    Graham Hutton has uploaded two sets of lectures from Nottingham University on his YouTube channel: Functional Programming in Haskell<\/a> and Advanced Functional Programming in Haskell<\/a>.<\/p>\n

    The first course covers functional programming, basic Haskell syntax, and how to work with types, type classes, and functions. The second course shows how to create a sudoku solver in Haskell and covers functors and monads. <\/p>\n

    The lectures are clear and well-polished, and you can\u2019t go wrong with spending your time watching them. <\/p>\n

    \n
    \n <\/a>
    \n \ud83d\udcfd\ufe0f Haskell videos
    \n<\/h2>\n

    If you\u2019re searching for something more to watch on YouTube, I have your back. <\/p>\n

    \n
    \n <\/a>
    \n Haskell for Imperative Programmers
    \n<\/h3>\n

    Haskell for Imperative Programmers<\/a> is a fantastic series of short videos that teaches functional programming in Haskell. <\/p>\n

    In comparison to the courses listed above, the sequencing of topics feels a bit arbitrary, so I wouldn\u2019t try to learn Haskell just by watching these series. But the individual videos are very well made. If you need to understand a specific topic, they are a good option.<\/p>\n

    \n
    \n <\/a>
    \n Category Theory for Programmers
    \n<\/h3>\n

    Category Theory for Programmers<\/a> by Bartosz Milewski is a series of videos that covers basics of category theory \u2013 the place from which all the scary Haskell words such as functors, monads, etc. come from \u2013 in a programmer-friendly fashion. <\/p>\n

    It\u2019s an excellent playlist to watch if you\u2019re interested in these kinds of things, and it will help you better understand how Haskell works. At the same time, remember that you don\u2019t need to know category theory to write perfectly fine Haskell programs. <\/p>\n

    \n
    \n <\/a>
    \n \ud83d\udd25 Other resources
    \n<\/h2>\n

    \n
    \n <\/a>
    \n Exercism
    \n<\/h3>\n

    Exercism<\/a> is a great option to get used to writing code in Haskell or any other programming language they offer. The Haskell track has plenty of exercises to keep you occupied for a while, and you can get free help with your code from the awesome mentors that contribute their time there.<\/p>\n

    \n
    \n <\/a>
    \n CodinGame
    \n<\/h3>\n

    CodinGame<\/a> might appeal to competitively-minded programmers. It is similar in purpose to Exercism, but in contrast to Exercism, it is structured more like a game. You can compete with other players in timed code clashes, program bots, and, of course, do LeetCode-like problems as well. <\/p>\n

    \n
    \n <\/a>
    \n What I Wish I Knew When Learning Haskell
    \n<\/h3>\n

    What I Wish I Knew When Learning Haskell<\/a> is an extensive Haskell resource created by Stephen Diehl. It isn\u2019t really something I would read from start to finish, but if you need to reference something, I would see if it isn\u2019t covered there first. <\/p>\n

    \n
    \n <\/a>
    \n Monday Morning Haskell
    \n<\/h3>\n

    Monday Morning Haskell<\/a> is a website with a ton of resources, starting from beginner Haskell mini-courses to advanced real-world Haskell tutorials. <\/p>\n

    \n
    \n <\/a>
    \n Haskell for all
    \n<\/h3>\n

    While not entirely a beginner resource, Haskell for all<\/a> is an accessible blog by Gabriella Gonzalez that has been active for a decade already, and I highly recommend checking it out while learning Haskell. <\/p>\n

    For example, reading an article such as Electoral vote distributions are Monoids<\/a> while learning about monoids in Haskell can help you better understand the typeclass and how it can be used in real-life code. <\/p>\n
