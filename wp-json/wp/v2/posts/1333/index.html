\nI am working on a project and looking to implement realtime chat(from scratch, mostly) for mobile clients(flutter).<\/p>\n

I already have working experience with Go, and was wondering if Erlang has enough going for it in terms of concurrency and IPC that learning it and implementing the main chat service with it would help in keeping the system efficient and\/or provide useful abstractions that would be harder to implement\/use in Go.<\/p>\n

I’m planning to keep one socket per client and it is my perception that Erlang’s design would inherently make sending a message between concurrent instances managing different clients easier.<\/p>\n

I have not currently figured out an efficient way to do this. One way I imagine it could be done would be to have one central messageRouter component in the main thread that maintains open channels with all the goroutines serving the sockets. This component can then receive the messages and route them appropriately. However, keeping open channels in idle time seems way too costly to me, considering it’ll have to constantly listen on all the channels.<\/p>\n\n","protected":false},"excerpt":{"rendered":"
