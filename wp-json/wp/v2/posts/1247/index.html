Hey, Appwrite maintainer here \ud83d\udc4b<\/p>\n

TL;DR – Appwrite release v0.13 focuses on making our Storage service more scalable and Functions service more responsive, with a lot of added quality of life improvements. <\/p>\n

If it\u2019s your first time hearing about [Appwrite](https:\/\/github.com\/appwrite), it\u2019s an open-source Firebase alternative. We help you handle authentication, databases, file storage, cloud functions, and more with tons of SDKs for web, mobile, and server side languages.<\/p>\n

With Appwrite, you own all your data, you can self-host anywhere, sets up in minutes, and you can audit every line of code we ship \ud83d\ude42<\/p>\n

Here\u2019s everything new in Appwrite v0.13:<\/p>\n

Storage Buckets: Organize your files into buckets with shared permissions and rules for file type and size.<\/p>\n

Storage Adapters with S3 Support: Use S3 and S3 compatible services like DigitalOcean Spaces as the storage adapter for Appwrite.<\/p>\n

Support for (very) Big Files: Support large files through chunked uploads. Handling large files will no longer devour your memory. <\/p>\n

CLI v2.0: Appwrite CLI is now decoupled from Docker and provides a better experience when deploying to the Functions service.<\/p>\n

Faster Functions Service: Synchronous and asynchronous function execution with a much lower response time. <\/p>\n

New Functions Runtimes: Functions service now supports compiled languages like Rust and Swift, with improved dependency management.<\/p>\n

Even More OAuth Providers – OAuth with Notion, Stripe and Microsoft added + support for OAuth Refresh tokens.<\/p>\n

Check out everything we shipped this time around \ud83d\udc47
