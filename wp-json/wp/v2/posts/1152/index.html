While Wordle has caught Twitter feeds by storm, there\u2019s something I love about playing the New York Times Spelling Bee game each morning.<\/p>\n

It works by providing a set of letters, and you have to come up with as many words as you can that:<\/p>\n

    \n
  • Only use the provided letters<\/li>\n
  • Use the center letter at least once<\/li>\n
  • Are at least 4 characters long<\/li>\n<\/ul>\n

    Here\u2019s what a game might look like:<\/p>\n

    <\/a><\/p>\n

    Recently, I took it upon myself to build a solver for this game using Elixir. Most of the code is in this post, but if you’d like to see the whole project you can check it out on GitHub<\/a>.<\/p>\n

    \n
    \n <\/a>
    \n Defining success
    \n<\/h2>\n

    Because the words in Spelling Bee are curated and intentionally exclude obscure words, we should expect that this program will suggest words that are not included in the solution.<\/p>\n

    So I\u2019ll be judging the success of this program by the percentage of the solution words that are identified, ignoring suggested words that are not part of the official solution.<\/p>\n

    \n
    \n <\/a>
    \n Getting a list of words
    \n<\/h2>\n

    My approach was pretty simple. I needed to start with a list of English words, then filter based on the criteria of the game.<\/p>\n

    To get a list of English words, I used the word_list package that I created<\/a>.<\/p>\n

    \n
    \n <\/a>
    \n Starting the project
    \n<\/h2>\n

    Once I created my new project, I imported my word_list<\/code> package by adding it in the deps<\/code> array in mix.exs<\/code><\/p>\n

    \n
    defp<\/span> deps<\/span> do<\/span>\n  [<\/span>\n    {<\/span>:word_list<\/span>,<\/span> \"~> 0.1.0\"<\/span>}<\/span>\n  ]<\/span>\nend<\/span>\n<\/code><\/pre>\n
