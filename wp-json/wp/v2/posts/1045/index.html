We all have used Instagram or WhatsApp to have a healthy chat with our friends. Being a developer, I always wondered how this happens so fast without reloading. The answer is Web Sockets.<\/p>\n

\n
\n <\/a>
\n Web Sockets vs HTTP
\n<\/h2>\n

Why Web Sockets you might ask? Well what happens in HTTP is that a connection is established and once the request result process is completed, the connection is terminated.
So if we use this process for real time messaging, it would take some time to create new connections, thus making our chatting process slower. No one would like that, right?<\/p>\n

This is where Web Sockets come in. The major difference is that the connection once established, is not destroyed until there is no more need for it. <\/p>\n

\n
\n <\/a>
\n Pipelines in Web Sockets
\n<\/h3>\n

This might feel like one of those tech terminologies which scare us. But in simpler terms, pipelines in tech are same as real life pipelines. A connection through which things can be passed in any direction easily.
This is how Web Sockets work, through pipelines.<\/p>\n
