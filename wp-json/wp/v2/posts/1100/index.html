With the rise of the Metaverse and 3D games such as Battle Royale, the demand for immersive audio experiences in virtual environments is growing rapidly. Spatial audio, a technology that allows users to perceive the location and distance of a sound source around them in a virtual scene, is quickly becoming an essential part of creating immersive virtual experiences.<\/p>\n

In response to this rapidly growing demand for an immersive audio experience, we’ve added a Proximity Voice<\/strong> module to the ZEGOCLOUD Express Web SDK (since v2.10.0), which provides the following features:<\/p>\n

    \n
  • \nProximity voice chat<\/strong>: A form of voice chat in virtual spaces chat where users can only hear the voice of other users from within a certain proximity, and the volume of the sound changes according to the distance between the listener and the sound source.<\/li>\n
  • \nSpatial audio<\/strong>: Users in a virtual space can sense the position and distance of a sound source as they do when hearing a sound in the real world.<\/li>\n
  • \nTeam voice chat<\/strong>: Users can join a team and switch between the Team-only<\/strong> mode (the user’s voice can only be heard by other users in the same team) and the Everyone<\/strong> mode (the user’s voice can be heard by everyone in the room) as they wish. <\/li>\n<\/ul>\n

      \n An introduction to Web Audio API
      \n<\/h2>\n

      The Web Audio API can be used for many different audio operations. For example, it is often used to replace the <audio><\/code> tag to play audio on the web. In addition, it provides other audio-processing capabilities, such as audio volume adjustment, audio mixing, and audio spatialization.<\/p>\n

      The Web Audio API lets you perform audio operations inside an audio context<\/strong>, and has been designed to allow modular routing<\/strong>.
