Java Streams API was introduced in Java 8. Its aim is to provide a less verbose and concise way to carry out common operations on collections of objects.<\/p>\n

Although it can be hard to get used to, the Java Streams API is very powerful and can be used to implement complex algorithms.<\/p>\n

In this article, we will talk about some common use cases of the Java Streams API. <\/p>\n

For those of us not proficient with the Streams API, the goal should be to keep these use cases in mind and whenever we come across one of them, we should try to implement them using Streams rather than the traditional way.<\/p>\n

Let’s establish some basics first.<\/p>\n

    \n
  • \nstream()<\/code> – creates a stream from a collection<\/li>\n
  • \ncollect()<\/code> – collects the stream into an object. The object can be a collection, primitive, or a custom class.<\/li>\n
  • \nCollectors<\/code> – a class which provides (a lot of) static methods for collecting streams. We will use some of these below. Refer to the documentation for full reference.<\/li>\n<\/ul>\n

    These are the most popular stream operations and will be used in the rest of this tutorial.<\/p>\n
