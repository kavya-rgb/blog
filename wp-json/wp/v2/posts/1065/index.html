Software development and testing are an ecosystem that works on the collective efforts of developers and testers. <\/p>\n

Every new addition or modification to the application has to be carefully tested before it is released to the customers or end-users.<\/p>\n

In order to have a robust, secure, and fully functional end application, it has to undergo a series of tests. <\/p>\n

There are multiple testing techniques involved in the whole process, however, Smoke and Sanity<\/a> are the first ones to be planned.<\/p>\n

Let us examine Smoke and Sanity testing in this article.<\/p>\n

Smoke testing essentially checks for the stability of a software build. It can be deemed as a preliminary check before the build goes for testing.<\/p>\n

Sanity testing is performed after a stable build is received and testing has been performed. It can be deemed as a post-build check, to make sure that all bugs have been fixed. <\/p>\n

It also makes sure that all functionalities are working as per the expectations. <\/p>\n

Let us first take a quick look at the details of both types of testing, and then we can move onto identifying the differences between them. There is a thin line between them, and people tend to confuse one for another.<\/p>\n

What is smoke testing?<\/p>\n

smoke testing<\/p>\n

Smoke testing, also known as build verification testing, is performed on initial builds before they are released for extensive testing. <\/p>\n
